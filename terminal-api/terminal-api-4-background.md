## 重複キー破棄

- `transactions` テーブルの `idempotencyKey` というカラムを NULL 埋めしている。
- 対象は `transactionAt` が１日以上経過していること。
- 処理の実行サイクルは１時間指定。  
  ただし、Android の WorkManager 機能で動作しているため、  
  実際の実行タイミングはOSリソースの都合により決まる。  
  概ね ± 10分 ほど誤差があるらしい。
